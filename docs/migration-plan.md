# 从飞书 CRM 迁移到 NAS CRM 的计划

## 阶段 1：先跑空系统

目标：

- NAS 上 CRM 能打开。
- 管理员账号可登录。
- 销售账号和角色可创建。
- 线索快录、客户新增、沟通记录、提醒规则能正常跑。

不导入真实客户数据。

## 阶段 2：小范围试用

选择 1-2 名销售试用一周：

- 每天录入新线索。
- 每次沟通后追加记录。
- 检查下次联系时间是否符合销售习惯。
- 检查公海提醒是否过严或过松。

## 阶段 3：导入历史数据

导入前先做三件事：

1. 从飞书导出客户和线索备份。
2. 在 NAS 上做数据库快照。
3. 先导入 20 条样例验证字段映射。

字段映射建议：

| 飞书字段 | NAS CRM 字段 |
|---|---|
| 客户编号 | customer_no |
| 客户名称 | name |
| 联系人 | contact_name |
| 联系电话 | phone |
| 微信号 | wechat |
| 邮箱 | email |
| 地区 | region |
| 线索来源 | source_channel |
| 客户类型 | customer_type |
| 客户需求 | demand |
| 客户负责人 | owner |
| 客户级别 | grade |
| 客户状态 | status |
| 最近跟进时间 | last_contact_at |
| 下次联系时间 | next_contact_at |

## 阶段 4：切换入口

飞书表格保留只读备份，新数据统一录入 NAS CRM。

## 阶段 5：接机器人

第一版可用飞书群机器人提醒。稳定后再做：

- 个人私聊提醒。
- CRM助手对话式录入线索。
- 语音转文字后自动生成沟通记录。
- 本地 AI 自动标签和客户分级建议。
